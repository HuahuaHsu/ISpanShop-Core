# 商品編輯圖片路徑修復說明

## ✅ 修復完成

已修復編輯商品時新增圖片導致舊圖片被刪除的問題。

---

## 🐛 問題診斷

### **問題現象**

編輯商品時新增一張圖片後重新送審，原本的舊圖片消失了。

**Console 輸出：**
```javascript
更新商品圖片: {
  newImages: 1,
  existingImages: 1,  // 明明有傳 existingImages
  totalImages: 2,
  originalCount: 1
}
```

但後端還是刪除了舊圖片。

### **根本原因**

**URL 格式不一致！**

| 位置 | 格式 | 範例 |
|------|------|------|
| 資料庫存的 | 相對路徑 | `/uploads/products/xxx.jpg` |
| 前端送出的 | 完整 URL | `https://localhost:7125/uploads/products/xxx.jpg` |

**流程分析：**

1. 後端回傳：`/uploads/products/xxx.jpg`（相對路徑）
2. 前端載入時轉成完整 URL：`https://localhost:7125/uploads/products/xxx.jpg`（方便顯示）
3. 編輯時送出 `existingImages`：`https://localhost:7125/uploads/products/xxx.jpg`（❌ 錯誤）
4. 後端比對資料庫：找不到匹配的路徑
5. 後端判斷：舊圖片都不保留，全部刪除

---

## 🔧 修復方案

### **1. 修正 existingImages 送出格式**

**位置：** `handleSubmit` 函式

**修復前：**
```typescript
for (const file of form.images) {
  if (file.raw) {
    formData.append('images', file.raw)
  } else if (file.url) {
    // ❌ 直接送出完整 URL
    formData.append('existingImages', file.url)
  }
}
```

**修復後：**
```typescript
for (const file of form.images) {
  if (file.raw) {
    // 新上傳的圖片
    formData.append('images', file.raw)
  } else if (file.url) {
    // 舊圖片的 URL，需要轉成相對路徑（跟資料庫格式一致）
    let imageUrl = file.url
    try {
      // 如果是完整 URL，只取路徑部分
      const url = new URL(imageUrl)
      imageUrl = url.pathname  // 例如：/uploads/products/xxx.jpg
    } catch {
      // 已經是相對路徑，不需要轉換
    }
    formData.append('existingImages', imageUrl)  // ✅ 送出相對路徑
  }
}
```

**關鍵改進：**
- ✅ 使用 `new URL(imageUrl).pathname` 提取路徑部分
- ✅ try-catch 處理已經是相對路徑的情況
- ✅ 確保送出的格式與資料庫一致

---

### **2. 修正新上傳圖片的打勾狀態**

**位置：** `handleImageChange` 函式

**問題：** 新上傳的圖片右上角沒有打勾圖示（✓）

**原因：** 舊圖片有打勾是因為 `status: 'success'`，新圖片沒有設定 status

**修復前：**
```typescript
const handleImageChange: UploadProps['onChange'] = (uploadFile, uploadFiles) => {
  // 驗證邏輯...
  
  // ❌ 沒有設定 status
}
```

**修復後：**
```typescript
const handleImageChange: UploadProps['onChange'] = (uploadFile, uploadFiles) => {
  // 驗證邏輯...
  
  // ✅ 設定上傳狀態為成功（顯示打勾圖示）
  if (uploadFile.status === 'ready') {
    uploadFile.status = 'success'
  }
}
```

```

  ↓

```

## 🔍 URL 解析範例

### **new URL().pathname 的作用**

```javascript
// 範例 1: 完整 URL
const url1 = new URL('https://localhost:7125/uploads/products/abc.jpg')
console.log(url1.pathname)  // "/uploads/products/abc.jpg" ✅

// 範例 2: 帶 query string
const url2 = new URL('https://localhost:7125/uploads/products/xyz.jpg?v=123')

// 範例 3: 已經是相對路徑（會拋出錯誤）

  const url3 = new URL('/uploads/products/def.jpg')
} catch (e) {
  // ✅ catch 中不轉換，直接使用原始值
  console.log('/uploads/products/def.jpg')
}
```

## 📝 程式碼對比

### **handleSubmit 完整修復**

```typescript
// 2. 檢查圖片是否有變動
const newFiles = form.images.filter(f => f.raw)
const hasNewImages = newFiles.length > 0
const hasRemovedImages = form.images.length !== originalImageCount.value

// 只有在圖片真正有變動時才呼叫圖片 API
if (hasNewImages || hasRemovedImages) {
  const formData = new FormData()
  
  // 遍歷所有圖片
  for (const file of form.images) {
    if (file.raw) {
      // 新上傳的圖片
      formData.append('images', file.raw)
    } else if (file.url) {
      // 舊圖片的 URL，需要轉成相對路徑（跟資料庫格式一致）
      let imageUrl = file.url
      try {
        // 如果是完整 URL，只取路徑部分
        const url = new URL(imageUrl)
        imageUrl = url.pathname  // ✅ 例如：/uploads/products/xxx.jpg
      } catch {
        // 已經是相對路徑，不需要轉換
      }
      formData.append('existingImages', imageUrl)  // ✅ 送出相對路徑
    }
  }
  
  console.log('更新商品圖片:', {
    newImages: newFiles.length,
    existingImages: form.images.filter(f => !f.raw).length,
    totalImages: form.images.length,
    originalCount: originalImageCount.value,
  })
  
  await updateProductImages(productId.value, formData)
} else {
  console.log('圖片無變動，跳過圖片更新 API')
}
```

## 🎯 關鍵要點

### **為什麼需要轉換？**

| 階段 | URL 格式 | 原因 |
|------|---------|------|
| 載入顯示 | 完整 URL | 前端 `<img>` 需要完整路徑才能顯示 |
| 送出更新 | 相對路徑 | 後端資料庫存的是相對路徑，需要匹配 |

### **載入時：相對路徑 → 完整 URL**
```typescript
// loadProductData
url: url.startsWith('http') ? url : `https://localhost:7125${url}`
```

### **送出時：完整 URL → 相對路徑**
```typescript
// handleSubmit
const url = new URL(imageUrl)
imageUrl = url.pathname
```
