# 商品刪除功能說明

## ✅ 功能完成

已在賣家商品列表頁面加入刪除功能，並根據商品狀態控制刪除按鈕的顯示。

---

## 🎯 功能需求

### **1. 刪除按鈕顯示規則**

| 商品狀態 | status 值 | 可否刪除 | 說明 |
|---------|----------|---------|------|
| 未上架 | 0 (draft) | ✅ 可刪除 | 尚未刊登的商品 |
| 已上架 | 1 (on) | ❌ 不可刪除 | 正在販售中，需先下架才能刪除 |
| 審核中 | 2 (review) | ❌ 不可刪除 | 正在審核中，不允許刪除 |
| 已退回 | 3 (rejected) | ✅ 可刪除 | 審核退回的商品 |

### **2. 刪除流程**

1. 點擊商品卡片右上角「...」更多選單
2. 選擇「刪除」選項
3. 彈出確認訊息：**「確定要刪除此商品嗎？刪除後可在「違規/刪除」中查看」**
4. 點擊「確定刪除」
5. 呼叫 `DELETE /api/seller/products/{id}` (軟刪除)
6. 刪除成功後：
   - 顯示成功訊息：**「商品已刪除」**
   - 重新載入商品列表
   - 商品從原 tab 消失
   - 商品出現在「違規/刪除」tab

```typescript
/**
 * 刪除商品（軟刪除）
 * DELETE /api/seller/products/{id}
 */
export async function deleteSellerProduct(
  id: number,
): Promise<ApiResponse<unknown>> {
  const response = await request.delete<ApiResponse<unknown>>(
    `/api/seller/products/${id}`,
  )
  return response.data
}
```

### **3. 刪除函式**

```typescript
// ── 刪除商品 ──────────────────────────────────────────────────────
async function handleDeleteProduct(product: SellerProduct): Promise<void> {
  try {
    await ElMessageBox.confirm(
      '確定要刪除此商品嗎？刪除後可在「違規/刪除」中查看',
      '刪除確認',
      {
        confirmButtonText: '確定刪除',
        cancelButtonText: '取消',
        type: 'warning',
      }
    )
    
    await deleteSellerProduct(product.id)
    ElMessage.success('商品已刪除')
    
    // 重新載入列表
    await loadProducts()
  } catch (error) {
    if (error !== 'cancel') {
      console.error('刪除商品失敗:', error)
      ElMessage.error('刪除失敗，請稍後再試')
    }
  }
}
```

```typescript
async function handleCardCommand(cmd: string, product: SellerProduct): Promise<void> {
```

## 🎨 各狀態的選單項目

### **已上架 (status = 'on')**

```
✅ 複製
✅ 即時預覽
✅ 下架
✅ 低庫存提醒
❌ 刪除（不顯示）
```

**原因：** 已上架的商品正在販售中，不允許直接刪除，需先下架。

---

### **未上架 (status = 'draft')**

```
✅ 複製
✅ 即時預覽
✅ 上架
✅ 低庫存提醒
✅ 刪除
```

**原因：** 未上架的商品可以自由管理，包括刪除。

---

### **已退回 (status = 'rejected')**

```
✅ 複製
❌ 即時預覽（不顯示）
❌ 上架/下架（不顯示）
❌ 低庫存提醒（不顯示）
✅ 刪除
```

**原因：** 已退回的商品不能上架，但可以複製或刪除。

---

### **審核中 (status = 'review')**

```
✅ 複製
✅ 即時預覽
❌ 上架/下架（不顯示）
❌ 低庫存提醒（不顯示）
❌ 刪除（不顯示）
```

**原因：** 審核中的商品不允許刪除，也不能上下架。

## ⚠️ 注意事項

### **1. 「違規/刪除」Tab 顯示**

後端 API 可能預設過濾掉 `IsDeleted=true` 的商品。

**如果「違規/刪除」tab 沒有顯示已刪除商品，需要：**

1. 確認後端 API 是否回傳已刪除的商品
2. 如果沒有，需要加一個參數：
   ```typescript
   GET /api/seller/products?includeDeleted=true
   ```
3. 或者在「違規/刪除」tab 呼叫不同的 API：
   ```typescript
   GET /api/seller/products/deleted
   ```

### **2. 軟刪除 vs 硬刪除**

當前實作為**軟刪除**（Soft Delete）：
- 商品不會從資料庫真正刪除
- 只是標記 `IsDeleted = true`
- 商品仍可在「違規/刪除」tab 查看
- 可能需要後續的「永久刪除」或「恢復」功能

### **3. 已上架商品的刪除流程**

如果要刪除已上架的商品：
1. 先點擊「下架」
2. 商品狀態變為「未上架」
3. 再點擊「刪除」
