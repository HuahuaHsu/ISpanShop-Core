### 1. 新增 API 函式 (src/api/product.ts)

```typescript
/**
 * 更新商品狀態（上架/下架）
 * PATCH /api/seller/products/{id}/status
 */
export async function updateProductStatus(
  id: number,
  status: number,
): Promise<ApiResponse<unknown>> {
  const response = await request.patch<ApiResponse<unknown>>(
    `/api/seller/products/${id}/status`,
    { status },
  )
  return response.data
}
```

**API 規格：**
- **端點**：`PATCH /api/seller/products/{id}/status`
- **請求**：`{ "status": 0 | 1 }`
  - `status = 0`：下架
  - `status = 1`：上架
- **回應**：標準 ApiResponse 格式

#### (2) 實作 handleToggleShelf 函式

```typescript
async function handleToggleShelf(product: SellerProduct): Promise<void> {
  // 只有已上架(status=1)或已下架(status=0)的商品可以切換
  if (product.status !== 'on' && product.status !== 'off') {
    ElMessage.warning('此商品狀態無法進行上下架操作')
    return
  }

  const newStatus: ProductStatus = product.status === 'on' ? 'off' : 'on'
  const newStatusNumber = newStatus === 'on' ? 1 : 0
  const actionText = newStatus === 'on' ? '上架' : '下架'

  try {
    // 確認對話框
    await ElMessageBox.confirm(
      `確定要${actionText}此商品嗎？`,
      '提示',
      {
        confirmButtonText: '確定',
        cancelButtonText: '取消',
        type: 'warning',
      }
    )

    // 呼叫 API
    await updateProductStatus(product.id, newStatusNumber)
    
    // 更新本地狀態
    product.status = newStatus
    
    ElMessage.success(`商品已${actionText}`)
  } catch (error) {
    if (error !== 'cancel') {
      console.error(`${actionText}失敗:`, error)
      ElMessage.error(`${actionText}失敗，請稍後再試`)
    }
  }
}
```

**功能特點：**
1. **狀態檢查**：只允許 `on` 或 `off` 狀態的商品切換
2. **確認對話框**：操作前彈出確認視窗
3. **API 呼叫**：使用 PATCH 更新狀態
4. **本地更新**：成功後立即更新前端顯示
5. **錯誤處理**：取消操作或失敗時顯示對應訊息

#### (3) 更新 UI 顯示條件

```vue
<el-dropdown-item
```

**顯示規則：**

- ❌ **待審核** (`status === 'review'`)：不顯示上下架按鈕
- ❌ **已退回** (`status === 'rejected'`)：不顯示上下架按鈕
- ❌ **草稿** (`status === 'draft'`)：不顯示上下架按鈕
- ❌ **已刪除** (`status === 'deleted'`)：不顯示上下架按鈕

## 🎬 操作流程

### 場景 1：下架已上架的商品

1. 使用者點擊商品卡片的「⋯ 更多」按鈕
2. 選單顯示「下架」選項（因為商品 `status === 'on'`）
3. 使用者點擊「下架」
4. 彈出確認對話框：「確定要下架此商品嗎？」
5. 使用者點擊「確定」
6. 前端呼叫：`PATCH /api/seller/products/{id}/status { status: 0 }`
7. 後端處理並回應成功
8. 前端更新商品狀態為 `off`
9. 顯示成功訊息：「商品已下架」
10. 商品卡片狀態徽章變更為「已下架」
11. 下次打開選單時顯示「上架」選項

### 場景 2：上架已下架的商品

1. 使用者點擊商品卡片的「⋯ 更多」按鈕
2. 選單顯示「上架」選項（因為商品 `status === 'off'`）
3. 使用者點擊「上架」
4. 彈出確認對話框：「確定要上架此商品嗎？」
5. 使用者點擊「確定」
6. 前端呼叫：`PATCH /api/seller/products/{id}/status { status: 1 }`
7. 後端處理並回應成功
8. 前端更新商品狀態為 `on`
9. 顯示成功訊息：「商品已上架」
10. 商品卡片狀態徽章變更為「已上架」

### 場景 3：嘗試操作待審核/已退回商品

1. 使用者點擊待審核或已退回商品的「⋯ 更多」按鈕
2. 選單中**不顯示**上架/下架選項
3. （如果用戶透過其他方式觸發）顯示警告：「此商品狀態無法進行上下架操作」

## 🔧 後端 API 需求

### PATCH /api/seller/products/{id}/status

**請求：**
```json
{
  "status": 0  // 0=下架, 1=上架
}
```

**回應（成功）：**
```json
{
  "success": true,
  "message": "商品狀態已更新"
}
```

**回應（失敗）：**
```json
{
  "success": false,
  "message": "更新失敗：[原因]"
}
```

**權限檢查：**
- 需要驗證 JWT token
- 只允許商品擁有者更新自己的商品
- 待審核/已退回的商品可能需要特殊處理（建議後端也檢查狀態）
