# 退回商品顯示功能說明

## ✅ 已完成的功能

### 1. 後端狀態對應
ProductListView.vue 已正確對應退回商品的狀態值：

```typescript
function mapStatusToKey(status: number): ProductStatus {
  switch (status) {
    case 1: return 'on'        // 已上架
    case 2: return 'review'    // 審核中
    case 3: return 'rejected'  // 審核退回 ⬅️ 退回商品
    default: return 'draft'    // 未上架
  }
}
```

### 2. Tab 篩選
已加入「已退回」Tab，位於導航列第 3 個位置：

```typescript
const level1Tabs = [
  { key: 'all',      label: '全部' },
  { key: 'on',       label: '架上商品' },
  { key: 'rejected', label: '已退回' },      // ⬅️ 退回商品專用 Tab
  { key: 'review',   label: '審核中' },
  { key: 'draft',    label: '未上架/尚未刊登' },
  { key: 'deleted',  label: '違規/刪除' },
]
```

點擊「已退回」Tab 會自動篩選出 status=3 的商品。

## 🔄 重新送審流程

1. **點擊「重新編輯」按鈕**  
   → 跳轉到 `/seller/products/{id}/edit`

2. **修改商品資訊**  
   → 根據退回原因修正問題（名稱、圖片、描述等）

3. **點擊「儲存並送審」**  
   → 商品重新進入審核流程
   → status 變為 `2`（審核中）
   → reviewStatus 變為 `3`（重新送審）

4. **等待後台審核**  
   → 通過：status 變為 `1`（已上架）
   → 再次退回：status 保持 `3`，更新 `rejectReason`

## 📝 注意事項

1. **rejectReason 欄位**
   - 如果後端沒有回傳 `rejectReason`，退回原因橫幅不會顯示
   - 建議後台管理員退回時務必填寫原因

2. **編輯頁面保留退回資訊**
   - 編輯頁面應該顯示退回原因（需要在 ProductEditView.vue 加入）
   - 讓賣家清楚知道為什麼被退回

3. **重新送審後的狀態**
   - 送審後 status 應變為 2（審核中）
   - reviewStatus 應變為 3（重新送審）
   - rejectReason 欄位保留，直到審核通過才清空
