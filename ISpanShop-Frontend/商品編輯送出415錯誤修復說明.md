# 商品編輯送出 415 錯誤修復說明

## ❌ 原始問題

### 錯誤訊息
```
PUT /api/seller/products/925
Status: 415 Unsupported Media Type
```

### 問題原因
編輯商品時使用了 **FormData (multipart/form-data)**，但後端 PUT endpoint 期待 **JSON (application/json)**。

```typescript
// ❌ 錯誤：編輯模式也用 FormData
const fd = new FormData()
fd.append('Name', form.name)
fd.append('CategoryId', String(form.categoryId))
// ...
await updateSellerProduct(productId.value, fd)  // 送出 multipart/form-data
```

### Console 輸出的問題
```javascript
BrandId: '(未選擇，不傳送)'  // ❌ 字串而非 null
```

---

## ✅ 修復方案

### 關鍵概念：新增和編輯使用不同的 Content-Type

| 操作 | HTTP 方法 | Content-Type | 原因 |
|------|----------|--------------|------|
| **新增商品** | POST | `multipart/form-data` | 需要上傳圖片檔案 |
| **編輯商品** | PUT | `application/json` | 只更新文字資料，不處理圖片 |

### 1. 修正 API 函式 (src/api/product.ts)

**修復前：**
```typescript
export async function updateSellerProduct(
  id: number,
  data: FormData,  // ❌ 強制使用 FormData
): Promise<ApiResponse<{ productId: number }>> {
  const response = await request.put(
    `/api/seller/products/${id}`,
    data,
    { headers: { 'Content-Type': 'multipart/form-data' } },  // ❌ 強制 multipart
  )
  return response.data
}
```

**修復後：**
```typescript
export async function updateSellerProduct(
  id: number,
  data: any,  // ✅ 接受任何類型，通常是 JSON 物件
): Promise<ApiResponse<{ productId: number }>> {
  const response = await request.put(
    `/api/seller/products/${id}`,
    data,  // ✅ axios 會自動根據資料類型設定 Content-Type
  )
  return response.data
}
```

**關鍵改變：**
- 移除 `FormData` 型別限制，改為 `any`
- 移除手動設定的 `Content-Type` header
- 讓 axios 自動判斷：
  - 傳入 JSON 物件 → 自動設定 `application/json`
  - 傳入 FormData → 自動設定 `multipart/form-data`

### 2. 修正 handleSubmit 函式 (ProductEditView.vue)

**修復前：**
```typescript
// ❌ 新增和編輯都用 FormData
const fd = new FormData()
fd.append('Name', form.name)
// ...

if (isEditMode.value) {
  await updateSellerProduct(productId.value, fd)  // ❌ 415 錯誤
} else {
  await createSellerProduct(fd)
}
```

**修復後：**
```typescript
if (isEditMode.value && productId.value) {
  // ===== 編輯模式：使用 JSON =====
  const updateData = {
    name: form.name,
    description: form.description || '',
    categoryId: form.categoryId,
    brandId: form.attributes.brandId || null,  // ✅ null 而非字串
    price: form.price || 0,
    stock: form.stock || 0,
    minPurchase: form.minPurchase || 1,
    specDefinitionJson: specsEnabled.value && form.specs.length > 0
      ? JSON.stringify(form.specs.map(s => ({ name: s.name })))
      : '[]',
  }

  console.log('送出資料 (編輯模式/JSON):', updateData)
  res = await updateSellerProduct(productId.value, updateData)
  
} else {
  // ===== 新增模式：使用 FormData =====
  const fd = new FormData()
  fd.append('Name', form.name)
  fd.append('Description', form.description)
  if (form.categoryId !== null) fd.append('CategoryId', String(form.categoryId))
  if (form.attributes.brandId !== null) fd.append('BrandId', String(form.attributes.brandId))
  
  // 上傳圖片...
  form.images.forEach((img) => {
    if (img.raw) fd.append('Images', img.raw)
  })
  
  console.log('送出資料 (新增模式/FormData):', {...})
  res = await createSellerProduct(fd)
}
```

### 3. 修正 BrandId 的值

**修復前：**
```typescript
BrandId: form.attributes.brandId ?? '(未選擇，不傳送)'  // ❌ 字串
```

**修復後：**
```typescript
brandId: form.attributes.brandId || null  // ✅ null
```

## 🔧 為什麼編輯不上傳圖片？

### 設計考量

1. **效能問題**
   - 每次編輯都上傳所有圖片會浪費頻寬
   - 圖片通常不會每次都改

2. **簡化邏輯**
   - 編輯時只更新文字資料（名稱、描述、價格等）
   - 圖片管理另外處理（未來可加入「編輯圖片」功能）

3. **後端架構**
   - POST endpoint 接受 multipart/form-data（包含圖片）
   - PUT endpoint 接受 JSON（只更新文字）
   - 分離關注點，程式碼更清晰

### 未來改進方向

如果需要編輯時也能更新圖片，有兩種方案：

**方案 A：圖片另外上傳**
```typescript
// 1. 先上傳新圖片
const uploadedImageUrls = await uploadImages(newImages)

// 2. 再更新商品資料（包含圖片 URLs）
await updateSellerProduct(id, {
  name: '...',
  imageUrls: uploadedImageUrls,
})
```

**方案 B：後端支援 multipart**
```typescript
// 後端改為接受 multipart/form-data
// 前端編輯時也用 FormData
const fd = new FormData()
fd.append('Name', form.name)
if (newImages.length > 0) {
  newImages.forEach(img => fd.append('Images', img))
}
await updateSellerProduct(id, fd)
```

目前採用方案 A 的簡化版（編輯時不處理圖片）。
