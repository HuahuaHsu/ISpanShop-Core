### 3. 新增商品資料載入函式

```typescript
async function loadProductData(): Promise<void> {
  if (!isEditMode.value || !productId.value) return
  
  loading.value = true
  try {
    const res = await getSellerProductDetail(productId.value)
    const product = res.data
    
    // 基本資訊
    form.name = product.name || ''
    form.description = product.description || ''
    form.categoryId = product.categoryId || null
    
    // 分類路徑
    if (product.categoryPath && product.categoryPath.length > 0) {
      form.categoryPath = product.categoryPath.map(c => c.name).join(' > ')
    }
    
    // 品牌
    if (product.brand) {
      form.attributes.brandId = product.brand.id
    }
    
    // 價格和庫存
    if (product.priceRange) {
      form.price = product.priceRange.min || 0
    }
    form.stock = product.totalStock || 0
    
    // 圖片
    if (product.images && product.images.length > 0) {
      form.images = product.images.map((img, idx) => ({
        uid: -(idx + 1),
        name: `image-${idx}`,
        url: img.url,
        status: 'success' as const,
      }))
    }
    
    // 規格和變體
    if (product.variants && product.variants.length > 0 && product.specs && product.specs.length > 0) {
      specsEnabled.value = true
      
      form.specs = product.specs.map(spec => ({
        name: spec.name,
        options: spec.options.map(optName => ({
          name: optName,
          image: null,
          imagePreview: null,
        })),
      }))
      
      variantData.value = product.variants.map(v => ({
        spec1: Object.values(v.specValues)[0] || '',
        spec2: Object.values(v.specValues)[1] || null,
        price: v.price || null,
        stock: v.stock || 0,
        sku: '',
      }))
    }
    
    ElMessage.success('商品資料載入成功')
  } catch (error) {
    console.error('載入商品資料失敗:', error)
    ElMessage.error('載入商品資料失敗，請稍後再試')
  } finally {
    loading.value = false
  }
}
```

### 5. 更新 handleSubmit 函式

區分新增和編輯模式：

```typescript
async function handleSubmit(publishNow: boolean): Promise<void> {
  // ... 驗證邏輯 ...
  
  let res: any
  let targetProductId: number | null = null

  if (isEditMode.value && productId.value) {
    // 編輯模式：PUT /api/seller/products/{id}
    res = await updateSellerProduct(productId.value, fd)
    targetProductId = productId.value
  } else {
    // 新增模式：POST /api/seller/products
    res = await createSellerProduct(fd)
    targetProductId = res.data?.productId || null
  }
  
  const successMsg = isEditMode.value 
    ? '商品已更新' 
    : (publishNow ? '商品已提交審核，請等待管理員審核' : '草稿已儲存')

}
```

## 🔄 資料流程

### 新增模式 (路由: /seller/products/new)
1. productId = null
2. isEditMode = false
3. 表單為空白
4. 提交時 → POST /api/seller/products

### 編輯模式 (路由: /seller/products/:id/edit)
1. productId = route.params.id
2. isEditMode = true
3. onMounted → 呼叫 GET /api/seller/products/{id} 載入資料
4. 自動填充表單欄位
5. 提交時 → PUT /api/seller/products/{id}

## ⚠️ 注意事項

### 1. 圖片處理
- **新增模式**：上傳所有圖片
- **編輯模式**：
  - 只上傳新增的圖片（有 `raw` 屬性的）
  - 保留原有圖片（只有 `url` 的）
  - 後端需要支援部分更新

### 2. 規格變體更新
- 目前只在新增模式下建立變體
- **編輯模式的變體更新待後端實作**
- 建議後端提供：
  - `PUT /api/seller/products/{id}/variants/{variantId}` 更新單一變體
  - `DELETE /api/seller/products/{id}/variants/{variantId}` 刪除變體

### 3. minPurchase 欄位
- 前端已支援，但後端 GET 介面可能沒回傳
- 需要後端在 ProductDetail 中加入 `minPurchase` 欄位

### 4. 動態屬性
- 前端已支援載入和顯示
- 需要後端在 GET /api/seller/products/{id} 回傳商品的屬性值

## 🧪 測試步驟

### 測試編輯功能
1. 登入賣家帳號
2. 進入「我的商品」頁面
3. 點擊任一商品的「編輯」按鈕
4. 確認頁面標題顯示「編輯商品」
5. 確認表單已自動填入商品資料（名稱、描述、分類、圖片等）
6. 修改部分欄位
7. 點擊「儲存」或「儲存並重新送審」
8. 確認更新成功並返回商品列表

### 測試退回商品重新編輯
1. 後台管理員退回一個商品
2. 前台賣家點擊「已退回」Tab
3. 點擊「重新編輯」按鈕
4. 確認商品資料已載入
5. 根據退回原因修改
6. 點擊「儲存並重新送審」
7. 確認商品進入審核中狀態

### 測試新增功能
1. 點擊「新增商品」按鈕
2. 確認頁面標題顯示「新增商品」
3. 確認表單為空白
4. 填寫商品資料
5. 點擊「儲存並送審」
6. 確認建立成功

## 🎯 總結

**(a) 編輯頁面能自動載入商品資料了嗎？**
✅ 是的！編輯模式下會在 onMounted 時自動呼叫 `getSellerProductDetail(productId)` 載入商品資料，並填充到表單中。

**(b) 後端 GET /api/seller/products/{id} 有回傳完整資料嗎？**
⚠️ 需要確認後端是否已實作此 API，並確認回應格式是否符合 ProductDetail 型別。如果後端尚未實作，需要：
- 實作 GET /api/seller/products/{id} endpoint
- 回傳完整商品資料（包含圖片、規格、變體、屬性等）

**(c) 頁面標題有區分新增/編輯嗎？**
✅ 是的！
- 新增模式：麵包屑顯示「新增商品」
- 編輯模式：麵包屑顯示「編輯商品」
- 按鈕文字也會根據模式動態變化

所有前端功能已完成！只需確認後端 API 是否就緒。
