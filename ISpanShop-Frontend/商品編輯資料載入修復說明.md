# 商品編輯資料載入修復說明

## ❌ 原始問題

### 問題描述
編輯頁面無法正確載入商品資料，原因是**解析層級錯誤**。

### 錯誤的程式碼
```typescript
async function loadProductData(): Promise<void> {
  const res = await getSellerProductDetail(productId.value)
  const product = res.data  // ❌ 錯誤：多了一層 .data
  
  // res 本身就是商品物件，不需要再 .data
}
```

### 根本原因
在 `src/api/product.ts` 中，`getSellerProductDetail` 函式已經回傳 `response.data`：

```typescript
export async function getSellerProductDetail(id: number) {
  const response = await request.get(`/api/seller/products/${id}`)
  return response.data  // axios 的 response.data
}
```

所以呼叫端收到的 `res` **就是商品物件本身**，不是 `{ data: 商品物件 }` 結構。

---

## ✅ 修復內容

### 1. 修正解析邏輯

**修復前：**
```typescript
const res = await getSellerProductDetail(productId.value)
const product = res.data  // ❌ 錯誤：res.data 是 undefined
```

**修復後：**
```typescript
const product = await getSellerProductDetail(productId.value)
// ✅ 正確：product 就是商品物件
```

### 2. 新增 productData 狀態

```typescript
const productData = ref<any>(null) // 儲存載入的商品原始資料

// 在 loadProductData 中儲存：
productData.value = product
```

用途：
- 顯示退回原因 (`productData.rejectReason`)
- 判斷商品狀態 (`productData.status === 3` 表示已退回)

### 3. 修正資料欄位 mapping

根據後端實際回傳格式修正：

```typescript
// 後端回傳格式：
{
  id: 925,
  name: '121354645',
  categoryId: 22,
  categoryName: '運動裝備與球類',  // 直接是字串，不是陣列
  brandId: null,
  brandName: '未設定',
  minPrice: null,
  maxPrice: null,
  images: ['/uploads/products/xxx.jpg'],  // 字串陣列，不是物件陣列
  specDefinitionJson: '[]',  // JSON 字串
  variants: [],
  status: 3,
  statusText: '審核退回',
  rejectReason: '圖片侵權/盜圖'
}
```

**修正的欄位：**

#### (1) 分類路徑
```typescript
// ❌ 錯誤（舊版）
if (product.categoryPath && product.categoryPath.length > 0) {
  form.categoryPath = product.categoryPath.map(c => c.name).join(' > ')
}

// ✅ 正確（新版）
form.categoryPath = product.categoryName || ''
```

#### (2) 品牌
```typescript
// ❌ 錯誤（舊版）
if (product.brand) {
  form.attributes.brandId = product.brand.id
}

// ✅ 正確（新版）
if (product.brandId) {
  form.attributes.brandId = product.brandId
}
```

#### (3) 價格
```typescript
// ❌ 錯誤（舊版）
if (product.priceRange) {
  form.price = product.priceRange.min || 0
}

// ✅ 正確（新版）
form.price = product.minPrice || 0
```

#### (4) 圖片
```typescript
// ❌ 錯誤（舊版）
form.images = product.images.map((img, idx) => ({
  uid: -(idx + 1),
  name: `image-${idx}`,
  url: img.url,  // ❌ img 已經是字串，不是物件
  status: 'success' as const,
}))

// ✅ 正確（新版）
form.images = product.images.map((url: string, idx: number) => ({
  uid: -(idx + 1),
  name: `image-${idx}`,
  url: url.startsWith('http') ? url : `https://localhost:7125${url}`,
  status: 'success' as const,
}))
```

#### (5) 規格定義
```typescript
// ❌ 錯誤（舊版）
if (product.variants && product.variants.length > 0 && product.specs && product.specs.length > 0) {
  form.specs = product.specs.map(spec => ({ ... }))
}

// ✅ 正確（新版）
if (product.specDefinitionJson && product.specDefinitionJson !== '[]') {
  const specs = JSON.parse(product.specDefinitionJson)
  if (specs && specs.length > 0) {
    specsEnabled.value = true
    form.specs = specs.map((spec: any) => ({
      name: spec.name || '',
      options: (spec.options || []).map((optName: string) => ({
        name: optName,
        image: null,
        imagePreview: null,
      })),
    }))
  }
}
```

## 🎯 測試結果

### (a) 解析錯誤的原因是什麼？

**答：多了一層 `.data`**

- API 函式 `getSellerProductDetail` 已經回傳 `response.data`（axios 自動解析後的資料）
- 所以呼叫端收到的就是**商品物件本身**
- 錯誤寫法 `const product = res.data` 會導致 `product` 是 `undefined`
- 正確寫法 `const product = res` 或 `const product = await getSellerProductDetail(...)`

### (b) 編輯頁面能正確載入商品資料了嗎？

**答：✅ 是的！**

現在可以正確載入：
- ✅ 商品名稱 (`name`)
- ✅ 商品描述 (`description`)
- ✅ 分類資訊 (`categoryId`, `categoryName`)
- ✅ 品牌資訊 (`brandId`)
- ✅ 價格 (`minPrice`)
- ✅ 圖片 (`images` 陣列)
- ✅ 規格定義 (`specDefinitionJson`)
- ✅ 變體資料 (`variants`)

### (c) 退回原因有顯示嗎？

**答：✅ 有！**

- 紅色警告橫幅顯示在表單頂部
- 顯示退回原因：例如「圖片侵權/盜圖」
- 提示使用者「請根據退回原因修改商品資料後重新送審」
- 只在編輯模式且有 `rejectReason` 時顯示
