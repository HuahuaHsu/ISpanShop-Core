## 1. 專案概述

ISpanShop-Core 是一個以 **ASP.NET Core** 建置的電商平台後台系統，採用多專案的 **分層架構（Layered Architecture）**，核心功能包含：

- **商品管理**：新增、編輯、上下架、審核商品
- **庫存管理**：庫存監控、安全庫存設定、低庫存警示
- **分類管理**：樹狀分類維護（主分類 / 子分類）
- **分類規格管理**：商品屬性規格定義與分類綁定
- **前台 RESTful API**：提供前端介面或行動裝置存取商品與庫存資料

整個方案（Solution）包含 **6 個專案**，每個專案各司其職，形成清晰的職責分離。

## 3. 主要組件及其功能

### 3.1 ISpanShop.Models — 資料模型層

**職責**：定義所有的資料庫實體（Entity）和資料傳輸物件（DTO），是整個應用程式的資料契約核心。

#### 3.1.1 EF Core 實體（`EfModels/`）

| 實體類別 | 說明 |
|---------|------|
| `User` | 系統使用者（帳號、Email、角色、是否為賣家） |
| `Role` | 角色（管理員、賣家、買家） |
| `MemberProfile` | 會員詳細資料（姓名、電話、點數、消費總額） |
| `MembershipLevel` | 會員等級（折扣率、最低消費門檻） |
| `Store` | 賣場資訊（賣場名稱、是否通過審核） |
| `Brand` | 品牌（名稱、Logo、排序） |
| `Category` | 商品分類（支援父子階層） |
| `CategorySpec` | 分類規格屬性（顏色、尺寸等） |
| `CategorySpecMapping` | 分類與規格的多對多關聯 |
| `CategorySpecOption` | 規格的選項值（例如：顏色 → 紅/藍/綠） |
| `Product` | 商品（名稱、描述、狀態、最低/最高價格） |
| `ProductVariant` | 商品規格（SKU、售價、庫存、安全庫存） |
| `ProductImage` | 商品圖片（URL、是否為主圖、排序） |
| `Cart` / `CartItem` | 購物車（每位使用者唯一一個購物車） |
| `Order` / `OrderDetail` | 訂單（訂單號、狀態、金額、收件資訊） |
| `OrderReview` / `ReviewImage` | 訂單評價與評價圖片 |
| `PaymentLog` | 付款記錄（綠界相關欄位） |
| `PointHistory` | 點數異動記錄 |
| `Address` | 使用者收貨地址 |
| `BlacklistRecord` | 黑名單紀錄 |
| `ChatMessage` | 聊天訊息（買賣家對話） |
| `LoginHistory` | 登入歷史 |
| `SensitiveWord` | 敏感詞庫（用於商品自動審核） |
| `SupportTicket` | 客服工單 |

**`ISpanShopDBContext`** 是繼承自 `DbContext` 的 EF Core 資料庫上下文，使用 SQL Server，並由 EF Core Power Tools 自動產生。

#### 3.1.2 資料傳輸物件（`DTOs/`）

DTO 用於在各層之間傳遞資料，避免直接暴露資料庫實體。主要的 DTO 如下：

| DTO 類別 | 用途 |
|---------|------|
| `ProductListDto` | 商品列表顯示（簡化資訊） |
| `ProductDetailDto` | 商品詳細資訊（含規格、圖片） |
| `ProductCreateDto` | 前台賣家新增商品 |
| `ProductAdminCreateDto` | 管理員後台新增商品 |
| `ProductUpdateDto` | 更新商品基本資料 |
| `ProductReviewDto` | 待審核商品資訊 |
| `ProductVariantDetailDto` | 商品規格詳情 |
| `ProductVariantCreateDto` | 新增商品規格 |
| `ProductVariantUpdateDto` | 更新商品規格 |
| `ProductSearchCriteria` | 商品搜尋條件（含分頁） |
| `CategoryDto` | 分類資訊（用於下拉選單） |
| `CategorySpecDto` | 規格屬性資訊 |
| `InventoryListDto` | 庫存列表顯示 |
| `InventorySearchCriteria` | 庫存搜尋條件 |
| `PagedResult<T>` | 泛型分頁結果容器 |

#### 3.1.3 DataSeeder

`DataSeeder.cs` 在應用程式啟動時執行，負責：
- 填充初始測試資料（分類、商品、使用者等）
- 確保資料庫有足夠的「待審核商品」供測試使用（`EnsurePendingProductsAsync`）

### 3.2 ISpanShop.Repositories — 資料存取層

**職責**：封裝所有與資料庫的直接互動，使用 **Repository Pattern**，讓上層 Service 無需了解 EF Core 細節。

每個 Repository 都有對應的介面（Interface），方便測試與替換。

#### 主要 Repository

**`ProductRepository` / `IProductRepository`**
- `AddProduct()` — 新增商品（含 Variants、Images）
- `GetAllProducts()` — 取得所有商品
- `GetProductsPaged()` — 分頁 + 多條件篩選商品
- `GetPendingProducts()` — 取得待審核商品（Status = 2）
- `GetProductById()` — 取得單一商品詳情（含 Include 相關資料）
- `UpdateProductStatus()` — 更新商品狀態
- `ApproveProduct()` — 核准商品（Status → 1）
- `RejectProduct()` — 退回商品（Status → 3，記錄退回原因）
- `SoftDeleteProduct()` — 軟刪除（IsDeleted = true）
- `UpdateBatchStatusAsync()` — 批次更新上下架狀態
- `IsSkuExists()` — 檢查 SKU 是否重複
- 規格相關：`AddVariant()`、`UpdateVariant()`、`SoftDeleteVariant()`

**`InventoryRepository` / `IInventoryRepository`**
- `GetVariantsPaged()` — 分頁 + 篩選庫存列表
- `GetLowStockCount()` — 低庫存（庫存 < 安全庫存）數量
- `GetZeroStockCount()` — 零庫存數量
- `UpdateStock()` — 更新現有庫存
- `UpdateSafetyStock()` — 更新安全庫存
- `UpdateStockAndSafetyStock()` — 同時更新兩者

**`CategoryManageRepository` / `ICategoryManageRepository`**
- 分類的 CRUD 操作
- 取得分類樹狀結構
- 取得分類下的商品數量

**`CategorySpecRepository` / `ICategorySpecRepository`**
- 規格屬性的 CRUD
- 管理分類與規格的綁定關係（`CategorySpecMapping`）
- 管理可篩選規格（IsFilterable）

### 3.3 ISpanShop.Services — 商業邏輯層

**職責**：實作商業規則，協調 Repository 完成複雜操作。Services 是應用程式的核心邏輯所在。

#### `ProductService` — 商品商業邏輯

重要商業邏輯說明：

**1. 自動審核機制（`AutoReviewProduct`）**

商品新增時，系統自動判定初始狀態：
```
含敏感詞（高仿、毒品等）→ Status = 3（自動退回）
描述過短（< 10字）      → Status = 2（待手動審核）
正常                    → Status = 1（自動上架）
```

**2. SKU 自動生成（`GenerateUniqueSku`）**

若賣家未提供 SKU，系統自動產生格式為 `XXXXXXXX-時間戳` 的唯一代碼。

**3. 價格區間計算**

新增商品時，從所有規格（Variants）中自動計算並寫入 `MinPrice` 和 `MaxPrice`。

**4. 管理員特殊路徑**

管理員透過 `CreateProductAdmin()` 新增商品，**略過審核機制**，直接設定 Status = 1（上架），並自動建立預設「標準版」規格。

#### `InventoryService` — 庫存商業邏輯

- 分頁取得庫存列表（帶篩選：關鍵字、分類、狀態、庫存範圍）
- 計算庫存統計數字（總數、低庫存、零庫存、正常）
- 調整庫存與安全庫存

#### `CategoryManageService` — 分類管理邏輯

- 取得樹狀分類（父分類下有子分類集合）
- 新增/更新/刪除分類（刪除前檢查：有商品或有子分類時拒絕刪除）
- 控制分類顯示狀態（IsVisible）與排序

#### `CategorySpecService` — 規格屬性邏輯

- 管理分類規格屬性（含選項）
- 切換規格與分類的綁定（`ToggleBinding`）
- 切換某規格是否可作為篩選條件（`ToggleFilterable`）

### 3.4 ISpanShop.MVC — 後台管理介面

**職責**：提供管理員使用的後台 Web 介面，採用 ASP.NET Core MVC 模式（Controller + View）。

#### Controllers（控制器）

| 控制器 | 路由前綴 | 主要功能 |
|--------|---------|---------|
| `HomeController` | `/Home` | 首頁儀表板 |
| `ProductsController` | `/Products` | 商品列表、審核、新增、編輯、刪除 |
| `InventoryController` | `/Inventory` | 庫存監控儀表板 |
| `CategoryManageController` | `/CategoryManage` | 分類樹狀管理（AJAX 操作） |
| `CategorySpecsController` | `/CategorySpecs` | 規格屬性 CRUD、分類規格矩陣綁定 |
| `CategoryBindingController` | `/CategoryBinding` | 分類規格綁定（另一個視圖） |

#### Views（視圖）

視圖位於 `Views/` 資料夾，使用 Razor 語法（`.cshtml`）。支援 Bootstrap 為主的 UI 框架。

主要 View 資料夾：
- `Views/Products/` — 商品相關頁面（Index, Details, PendingReview, Create, Edit, AddVariant, EditVariant）
- `Views/Inventory/` — 庫存監控頁面
- `Views/CategoryManage/` — 分類管理頁面
- `Views/CategorySpecs/` — 規格屬性頁面
- `Views/Shared/` — 共用佈局（`_Layout.cshtml`）

#### MVC ViewModel

`Models/ViewModels/` 資料夾中定義了 ViewModel，作為 View 的資料容器（不直接使用 DTO 或 Entity）。

#### 圖片上傳處理

`ProductsController` 中的 `HandleImageUploadAsync()` 方法：
- 支援 `.jpg`、`.jpeg`、`.png`、`.webp` 格式
- 限制單檔最大 5 MB
- 儲存至 `wwwroot/uploads/products/` 目錄
- 以 GUID + 原始副檔名作為新檔名

### 3.5 ISpanShop.WebAPI — 前台 RESTful API

**職責**：提供 JSON 格式的 RESTful API，供前台網站或行動 App 呼叫。

#### Controllers（API 控制器）

| 控制器 | 路由 | 主要功能 |
|--------|------|---------|
| `ProductsApiController` | `POST /api/products` | 前台賣家新增商品 |
| `SellerInventoryController` | `GET/PUT /api/seller/inventory` | 賣家庫存管理 API |

#### Swagger / OpenAPI

開發環境啟動後，可直接在 `http://localhost:{port}/` 瀏覽 Swagger UI，查看所有 API 文件與測試端點。

#### CORS 設定

目前開發環境允許所有來源（`AllowAnyOrigin`），上線前需改為指定前台網域。

#### 全域例外處理 Middleware

`ExceptionHandlingMiddleware`：
- 捕捉所有未處理的例外
- 統一回傳 HTTP 500 + JSON 錯誤格式：`{ "message": "伺服器發生未預期的錯誤..." }`
- 記錄完整的錯誤 Log（含 HTTP 方法與路徑）

## 5. 數據流程

### 5.1 後台管理員新增商品流程

```
管理員在 MVC 後台填寫表單
           │
           ▼
ProductsController.Create() [POST]
  ├── 驗證 ModelState
  ├── 處理圖片上傳 (HandleImageUploadAsync)
  │    ├── 驗證副檔名與檔案大小
  │    └── 儲存至 wwwroot/uploads/products/
  └── 呼叫 _productService.CreateProductAdmin(dto)
               │
               ▼
       ProductService.CreateProductAdmin()
         ├── 建立 Product Entity（Status = 1，直接上架）
         ├── 自動產生 SKU（GUID 前8碼 + 時間戳）
         ├── 建立預設「標準版」規格
         └── 呼叫 _productRepository.AddProduct(product)
                         │
                         ▼
               ProductRepository.AddProduct()
                 ├── context.Products.Add(product)
                 └── context.SaveChanges()
                         │
                         ▼
                   資料寫入 SQL Server
```

### 5.2 前台賣家新增商品流程（含自動審核）

```
賣家透過前台介面提交商品（FormData）
           │
           ▼
ProductsApiController.CreateProduct() [POST]
  └── 呼叫 _productService.CreateProduct(dto)
               │
               ▼
       ProductService.CreateProduct()
         ├── AutoReviewProduct(name, description)
         │    ├── 含敏感詞 → Status = 3（退回）
         │    ├── 描述 < 10字 → Status = 2（待審核）
         │    └── 正常 → Status = 1（直接上架）
         ├── 計算 MinPrice / MaxPrice
         ├── 為每個 Variant 自動產生或驗證 SKU
         └── 呼叫 _productRepository.AddProduct(product)
```

### 5.3 庫存查詢與調整流程（WebAPI）

```
前台呼叫 GET /api/seller/inventory?keyword=xxx&status=low
           │
           ▼
SellerInventoryController.GetList()
  ├── 解析 Query Parameters → InventorySearchCriteria
  └── 呼叫 _inventoryService.GetInventoryPaged(criteria)
               │
               ▼
       InventoryService.GetInventoryPaged()
         └── 呼叫 _inventoryRepository.GetVariantsPaged(criteria)
                         │
                         ▼
               InventoryRepository.GetVariantsPaged()
                 ├── LINQ 查詢 ProductVariants
                 ├── Include Product/Store/Category
                 ├── 套用 keyword/category/stock 篩選
                 ├── 套用排序
                 └── 分頁（Skip/Take）
                         │
                         ▼
                回傳 PagedResult<InventoryListDto>
                         │
                         ▼
              回傳 JSON: PagedResultDto<InventoryItemDto>
```

### 5.4 商品審核流程

```
管理員開啟「待審核商品」列表
           │
ProductsController.PendingReview() [GET]
  └── _productService.GetPendingProducts()
      → 查詢 Status == 2 的商品

管理員點擊「核准」
           │
ProductsController.ApproveProduct() [POST, AJAX]
  └── _productService.ApproveProduct(id)
      └── _productRepository.ApproveProduct(id)
          → UPDATE Status = 1 WHERE Id = {id}

管理員點擊「退回」（含退回原因）
           │
ProductsController.RejectProduct() [POST, AJAX]
  └── _productService.RejectProduct(id, reason)
      └── _productRepository.RejectProduct(id, reason)
          → UPDATE Status = 3, RejectReason = {reason}
```

## 6. 各模組如何協作

### 6.1 依賴注入（DI）

兩個應用程式（MVC 和 WebAPI）都在 `Program.cs` 中向 DI 容器登記服務。所有 Service 和 Repository 都以 **Scoped 生命週期** 注入（每個 HTTP 請求建立一個新實例）。

**MVC Program.cs 中的服務登記：**
```csharp
builder.Services.AddDbContext<ISpanShopDBContext>(...);
builder.Services.AddScoped<IProductService, ProductService>();
builder.Services.AddScoped<IProductRepository, ProductRepository>();
builder.Services.AddScoped<ICategorySpecRepository, CategorySpecRepository>();
builder.Services.AddScoped<CategorySpecService>();
builder.Services.AddScoped<ICategoryManageRepository, CategoryManageRepository>();
builder.Services.AddScoped<CategoryManageService>();
builder.Services.AddScoped<IInventoryRepository, InventoryRepository>();
builder.Services.AddScoped<IInventoryService, InventoryService>();
```

**WebAPI Program.cs 中的服務登記：**
```csharp
builder.Services.AddDbContext<ISpanShopDBContext>(...);
builder.Services.AddScoped<IProductRepository, ProductRepository>();
builder.Services.AddScoped<IInventoryRepository, InventoryRepository>();
builder.Services.AddScoped<IProductService, ProductService>();
builder.Services.AddScoped<IInventoryService, InventoryService>();
```

> **重要**：兩個專案共用同一個 `ISpanShopDBContext`（連接同一個資料庫）、同一套 Repository 和 Service，確保邏輯一致。

### 6.2 介面隔離原則（Interface Segregation）

每個 Repository 和 Service 都有對應的 **介面（Interface）**：

```
IProductService ← ProductService ← 注入至 ProductsController (MVC)
                                  ← 注入至 ProductsApiController (WebAPI)

IProductRepository ← ProductRepository ← 注入至 ProductService
```

這樣設計的好處：
- **可測試性**：可用 Mock 物件替換真實實作進行單元測試
- **解耦合**：Controller 只依賴介面，不直接依賴具體實作

### 6.3 兩個前端入口點的分工

| 功能 | MVC 後台 (`ISpanShop.MVC`) | WebAPI (`ISpanShop.WebAPI`) |
|------|--------------------------|---------------------------|
| 商品管理 | ✅ 完整 CRUD、審核、批次上下架 | ✅ 前台賣家新增商品 |
| 庫存管理 | ✅ 庫存監控（唯讀儀表板） | ✅ 賣家庫存調整 API |
| 分類管理 | ✅ 完整樹狀 CRUD | ❌ |
| 規格管理 | ✅ 規格定義與分類綁定 | ❌ |
| API 文件 | ❌ | ✅ Swagger UI |

## 8. 商品狀態機

商品的 `Status` 欄位（tinyint）控制商品在系統中的生命週期：

```
                    管理員手動下架
                    ┌──────────────┐
                    ▼              │
  賣家提交  →  [2] 待審核  ──────▶ [1] 上架中 ──────▶ [0] 下架
  (描述不足)                核准    │                        ▲
                            ↑      │ 管理員下架              │
  賣家提交  →  [1] 直接上架  │      ▼                        │
  (正常)      (管理員新增)   │  [3] 審核退回                  │
                            │      │                        │
  賣家提交  →  [3] 自動退回  │      └── 賣家修改重新提交 ────▶ │
  (含敏感詞)  (自動審核)
```

| 狀態值 | 說明 | 觸發條件 |
|-------|------|---------|
| `0` | 下架 | 管理員手動下架 |
| `1` | 上架中 | 審核核准 / 管理員直接新增 / 正常賣家提交 |
| `2` | 待審核 | 賣家提交但描述過短 |
| `3` | 審核退回 | 含敏感詞（自動）或管理員手動退回 |

## 10. 重要設計模式

### 10.1 泛型分頁結果（`PagedResult<T>`）

系統統一使用 `PagedResult<T>` 封裝所有分頁查詢結果，提供：
- `Data` — 當頁資料集合
- `TotalCount` — 資料總筆數
- `TotalPages` — 總頁數
- `CurrentPage` — 目前頁碼
- `PageSize` — 每頁筆數

呼叫方式：
```csharp
var result = PagedResult<T>.Create(data, totalCount, currentPage, pageSize);
```

### 10.2 軟刪除（Soft Delete）

`Product` 和 `ProductVariant` 不實際刪除資料庫記錄，而是設置 `IsDeleted = true`。查詢時需過濾：

```csharp
// 商品查詢排除已軟刪除
var variants = product.ProductVariants?.Where(v => v.IsDeleted != true)
```

### 10.3 規格 JSON 格式

**`SpecDefinitionJson`**（商品的規格維度定義）：
```json
[
  { "name": "顏色", "options": ["紅", "藍", "綠"] },
  { "name": "尺寸", "options": ["S", "M", "L", "XL"] }
]
```

**`SpecValueJson`**（規格的具體值）：
```json
{ "顏色": "紅", "尺寸": "L" }
```

### 10.4 圖片儲存策略

| 來源 | 處理方式 |
|------|---------|
| 上傳圖片檔 | 儲存至 `wwwroot/uploads/products/{GUID}.{ext}`，DB 存相對路徑 |
| 外部圖片 URL | 直接儲存 URL 字串 |

```bash
cd ISpanShop.MVC

```
瀏覽 `https://localhost:{port}/`

**WebAPI**：
```bash
cd ISpanShop.WebAPI
dotnet run
```
瀏覽 `https://localhost:{port}/`（Swagger UI 在根路徑）

### 11.4 啟動時自動執行的任務

MVC 應用程式啟動時（`Program.cs`），會自動執行以下初始化工作：

1. **`DataSeeder.SeedAsync()`** — 填充基礎測試資料
2. **`DataSeeder.EnsurePendingProductsAsync()`** — 確保有 15 筆待審核商品
3. **動態 Schema 更新** — 自動檢查並新增 `Products.IsDeleted` 欄位（軟刪除）
4. **動態 Schema 更新** — 自動檢查並新增 `Categories.NameEn` 欄位（英文名稱）
5. **資料清理** — 移除商品名稱中錯誤加入的 `[待審核]` 前綴

> ⚠️ **注意**：步驟 3 和 4 使用 `ExecuteSqlRawAsync` 直接執行 DDL，屬於臨時的資料庫遷移補丁，正式環境建議改用 EF Core Migration。
