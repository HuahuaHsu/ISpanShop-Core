  「非同步影像上傳」的實作位置與方式：

  這項技術主要應用在「賣家商品管理」與「買家退貨申請」兩個模組中。

  1. 賣家商品管理 (Seller Product Images)
   * 實作位置： ISpanShop.MVC\Controllers\Api\Products\SellerProductsApiController.cs 中的 CreateProduct 方法。
   * 實作方式：
       * 後端： 使用 [ApiController] 標記的 SellerProductsApiController，透過 [FromForm] 接收 CreateProductRequest。其中的 Images 屬性是一個
         List<IFormFile>。
       * 技術特點： 雖然 CreateProduct 是一個整體的 POST 請求，但在現代前端架構中，這類 ApiController 通常配合 FormData 與 fetch/axios 進行 AJAX
         非同步提交。這樣使用者在點擊「提交」後，頁面不需重新整理即可完成包含多張大圖的上傳與商品建立。

  2. 買家退貨申請 (Return Request Image)
   * 實作位置： ISpanShop.MVC\Areas\Admin\Controllers\Orders\OrderTrackingController.cs 中的 ApplyReturn 方法。
   * 實作方式：
       * 核心代碼：
   1         using (var stream = new FileStream(filePath, FileMode.Create))
   2         {
   3             await evidence.CopyToAsync(stream); // 非同步寫入檔案流
   4         }
       * 技術細節： 這裡使用了 .NET 的 await CopyToAsync 技術。這意味著在高併發的上傳情境下，伺服器線程不會被磁碟 I/O
         阻塞，能有效提升伺服器的回應能力與上傳效率。
