## 問題：商品審核中心側邊詳情面板缺少審核操作按鈕

目前商品審核中心點擊商品展開右側詳情面板後，面板內只顯示商品資訊，沒有「核准上架」和「退回」按鈕。左側列表在精簡模式下又隱藏了操作欄，導致無法進行審核操作。

### 修復方式
在右側詳情面板的底部加入固定的操作按鈕區：

1. 按鈕區固定在面板底部（position: sticky; bottom: 0），不隨內容滾動而消失
2. 按鈕區有白色背景和頂部陰影，跟上方內容區分
3. 按鈕區內容根據商品狀態動態顯示：

待審核狀態：
- 「核准上架」按鈕（綠色）
- 「退回」按鈕（紅色）

重新申請審核狀態：
- 「核准上架」按鈕（綠色）
- 「再次駁回」按鈕（紅色）
- 額外顯示原始下架原因（讓管理員對照）

已核准/已退回狀態：
- 不顯示操作按鈕，改為顯示審核結果資訊（審核人、審核時間、原因等）

強制下架狀態（如果從商品總覽跳過來看的話）：
- 顯示「已強制下架」狀態資訊

4. 點擊「核准上架」：
   - 彈出確認 Modal（不要用原生 confirm）
   - 確認後呼叫 API
   - 成功後該商品從列表移除
   - 自動選中列表中的下一筆商品
   - 更新頂部的待審核數字

5. 點擊「退回」：
   - 彈出 Modal，必須填寫退回原因（必填，不限字數）
   - 提供快捷選項：商品描述不實、圖片不清晰、價格異常、分類錯誤、其他
   - 確認後呼叫 API
   - 成功後同樣從列表移除並選中下一筆

6. 如果審核完當前列表最後一筆，自動載入下一頁；如果已經沒有待審核商品，顯示空狀態「所有商品已審核完畢」

### 按鈕樣式
- 跟列表中原有的「核准上架」「退回」按鈕風格一致
- 按鈕要夠大，方便點擊（padding 充足）
- 兩個按鈕水平排列，中間有間距

### 除錯
- 確認審核操作後面板狀態正確更新
- 確認列表筆數和頂部統計數字即時更新
- 確認不會出現原生 confirm/alert
