# 商品種子資料修改說明

## 修改日期
2026-03-17

## 修改目的
讓系統初始化時，「商品審核中心」的「待審核」列表完全淨空（0 筆），以便專題展示時有豐富的已上架商品，而不是一堆待審核商品。

## 修改內容

### 檔案：`ISpanShop.Models\Seeding\DataSeeder.cs`

#### 1. 移除「將最後 15 筆設為待審核」的邏輯 (Line 132-136)

**修改前：**
```csharp
// 3. 將最後 15 筆設為「待審核」
foreach (var p in products.Skip(Math.Max(0, products.Count - 15)))
{
    p.Status = 2;
```

**修改後：**
```csharp
// 3. 所有商品預設為「已上架」狀態 (不再生成待審核商品)
// 備註：若需測試審核流程，請使用「商品總覽」的「生成測試用待審核商品」功能
```

#### 2. 修改商品建立時的狀態賦值 (Line 196-199)

**修改前：**

**修改後：**

#### 3. 停用 `EnsurePendingProductsAsync` 方法 (Line 472-482)

**修改前：**
```csharp
public static async Task EnsurePendingProductsAsync(ISpanShopDBContext context)
{
    var currentCount = context.Products.Count(p => p.Status == 2);
    if (currentCount >= 15)
    {
        Console.WriteLine($"ℹ️  待審核商品已有 {currentCount} 筆，無需補充。");
        return;
    }

    var needed = 15 - currentCount;
    var candidates = context.Products
        .Where(p => p.Status != 2 && p.IsDeleted != true)
        .Take(needed)
        .ToList();

    foreach (var p in candidates)
    {
        p.Status = 2;
        p.UpdatedAt = DateTime.Now;
    }

    await context.SaveChangesAsync();
    Console.WriteLine($"✅ 已補充 {candidates.Count} 筆待審核商品，目前共 {currentCount + candidates.Count} 筆");
}
```

**修改後：**
```csharp
/// <summary>
/// [已停用] 確保有足夠的待審核商品 (至少 15 筆)
/// 修改說明：為了讓系統初始化時「待審核」列表為空，此方法已停用
/// 若需測試審核流程，請使用「商品總覽」的「生成測試用待審核商品」功能
/// </summary>
public static async Task EnsurePendingProductsAsync(ISpanShopDBContext context)
{
    // 停用此功能：不再自動補充待審核商品
    Console.WriteLine("ℹ️  種子資料不再自動生成待審核商品，待審核列表保持淨空。");
    await Task.CompletedTask;
}
```

## 商品狀態說明

### Status（上下架狀態）
- **0**：已下架
- **1**：已上架 ✅ **[種子資料預設值]**
- **2**：待審核
- **3**：已退回
- **4**：強制下架

### ReviewStatus（審核狀態）
- **0**：待審核
- **1**：已審核通過 ✅ **[種子資料預設值]**
- **2**：已退回
- **3**：重新申請審核

## 修改效果

### ✅ 修改後的行為
1. **系統初始化時**：
   - 所有商品 `Status = 1`（已上架）
   - 所有商品 `ReviewStatus = 1`（已審核通過）
   - 「待審核」列表為空（0 筆）
   - 「商品總覽」列表有豐富的已上架商品

2. **庫存分配邏輯保持不變**：
   - 1/3 零庫存
   - 1/3 低庫存
   - 1/3 正常庫存

### ❌ 不受影響的部分
- 商品名稱、描述、價格
- 商品分類、品牌、賣場
- 商品圖片
- 規格變體（Variants）
- 庫存數量分配邏輯

## 測試審核流程

若需測試審核功能，請使用以下方式：

### 方法 1：使用「生成測試用待審核商品」功能
1. 前往「商品總覽」頁面
2. 點擊「生成測試用待審核商品」按鈕
3. 系統會生成 15 筆測試用待審核商品，涵蓋：
   - 5 筆乾淨商品（預期自動通過）
   - 5 筆高風險商品（預期自動退回）
   - 5 筆邊緣商品（預期待人工複審）

### 方法 2：手動建立待審核商品
1. 使用「新增商品」功能建立新商品
2. 新建的商品預設為「待審核」狀態
3. 可在「商品審核中心」進行審核

## 注意事項

⚠️ **資料庫清除後才會生效**

此修改只影響**新建立的種子資料**。若要看到效果，需要：

1. 刪除現有資料庫
2. 重新執行 Migrations
3. 重新啟動應用程式，觸發種子資料生成

或者：

1. 使用 SQL 指令將所有商品改為已上架狀態：
```sql
UPDATE Products 
SET Status = 1, 
    ReviewStatus = 1, 
    ReviewedBy = '系統預設', 
    ReviewDate = GETDATE()
WHERE Status = 2 OR ReviewStatus = 0;
```
