ISpanShop 系統邏輯說明文件：金流、點數、優惠券
================================================================================

本文件詳細說明專案中關於支付對接、點數折抵與優惠券核銷的核心程式碼運作邏輯。

--------------------------------------------------------------------------------
1. 金流系統 (Payment Integration - ECPay)
--------------------------------------------------------------------------------
專案主要透過綠界科技 (ECPay) 進行第三方支付。

● 核心類別：ISpanShop.Common/EcpayHelper.cs
● 關鍵功能：
  1. CheckMacValue 產生器 (GenerateCheckMacValue)：
     - 這是最核心的安全性邏輯。它會將訂單編號、金額、時間等參數按字母 A-Z 排序。
     - 前後串接 HashKey 與 HashIV。
     - 進行特殊的 URL 編碼 (CustomUrlEncode)，包含綠界要求的符號替換 (如 * 轉 %2a, ( 轉 %28)。
     - 最後透過 SHA256 加密並轉為大寫，作為綠界驗證資料來源的指紋。
  2. 字串清理 (CleanString)：
     - 由於綠界的參數不能包含特殊符號 (如 &、#)，此工具負責清理商品名稱或參數，避免 CheckMacValue 驗證失敗。

● 流程：
  [結帳頁] -> [產生訂單] -> [呼叫 EcpayHelper 封裝參數] -> [導向綠界] -> [綠界回傳支付結果] -> [驗證 CheckMacValue] -> [更新訂單狀態]。

2. 點數系統 (Point System - 蝦幣模擬)
--------------------------------------------------------------------------------
點數被視為一種資產，用於在結帳時進行現金折抵。

● 核心類別：ISpanShop.Services/Payments/CheckoutService.cs, PointService.cs
● 關鍵邏輯：
  1. 餘額檢查：在結帳 (Checkout) 過程中，系統呼叫 GetBalanceAsync 獲取會員當前餘額。
  2. 負值扣除：若使用者勾選「使用點數」，系統會計算 subtotal (小計) 與點數餘額的最小值。
  3. 點數更新：
     - 在訂單成立當下，系統會呼叫 UpdatePointsAsync 寫入一筆 PointHistory。
     - 變動金額 (ChangeAmount) 為負值 (例如：-50 點)。
     - 此時點數會被即時扣除，確保使用者不能在另一台裝置重複使用同一筆點數。

● 資料表關聯：
  - PointHistory 紀錄每筆變動來源 (如：Checkout_Deduction 代表消費扣除)。
  - MemberProfile 更新 CurrentPoints 欄位。

3. 優惠券系統 (Coupon System)
--------------------------------------------------------------------------------
優惠券模組支援複雜的校驗與狀態管理。

● 核心類別：ISpanShop.Services/Coupons/CouponService.cs
● 關鍵屬性：
  - CouponType: 1 (固定金額折抵，如折 $100)、2 (百分比折扣，如 9 折)。
  - UsageStatus: 0 (未使用)、1 (已使用)、3 (鎖定中/付款中)。

● 運作邏輯：
  1. 驗證 (ValidateCouponAsync)：
     - 檢查有效期限 (StartTime/EndTime)。
     - 檢查低消門檻 (MinimumSpend)。
     - 檢查是否僅限特定商品類別 (ApplyToAll)。
  2. 鎖定 (LockCouponAsync)：
     - 在下單但尚未付清前，將優惠券狀態標記為 3 (Locked)。
     - 這樣可以防止使用者在跳轉到金流頁面時，回頭又用同一張券下另一張單。
  3. 核銷 (UseCouponAsync)：
     - 當金流回傳付款成功時，將狀態改為 1 (Used)，並記錄 UsedAt 時間與 OrderId。
  4. 釋放 (ReleaseLockedCouponsAsync)：
     - 若訂單取消或超時未付款，系統會將狀態 3 的優惠券還原為 0，退還給使用者。

--------------------------------------------------------------------------------
4. 結帳時的金額計算順序 (Checkout Logic)
--------------------------------------------------------------------------------
在 CheckoutService 中，最終支付金額 (FinalAmount) 的計算公式如下：

  [商品小計 (Subtotal)]
  - [優惠券折抵 (CouponDiscount)] (優先計算，可能受最高折扣上限 MaximumDiscount 限制)
  = [剩餘應付金額 (RemainingAmount)]
  
  [剩餘應付金額]
  - [點數折抵 (PointDiscount)] (以剩餘應付金額為上限)
  + [運費 (ShippingFee)]
  = [最終付款金額 (FinalAmount)]
