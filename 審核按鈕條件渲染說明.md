# 審核按鈕條件渲染說明

## 修改日期
2026-03-17

## 問題描述
「核准上架」與「退回」審核按鈕在不該出現的頁面也會顯示，例如：
- 從「商品總覽」打開商品詳情
- 從「庫存監控」打開商品詳情
- 從「促銷活動管理」的「進行中」、「已拒絕」、「已結束」頁籤打開活動詳情

## 解決方案

### 1. Controller 層（傳遞審核模式標記）

#### 商品審核 (`ProductsController.cs`)
```csharp

{

    return PartialView("_ProductDetailsPartial", vm);
}
```

## 防護機制

### 雙重條件檢查
審核按鈕的顯示需要**同時滿足**兩個條件：

1. **來源頁面檢查** (`isReviewMode == true`)
   - ✅ 從「商品審核中心」的待審核/重新申請審核列表打開
   - ✅ 從「促銷活動管理」的待審核/重新申請審核列表打開
   - ❌ 從「商品總覽」打開
   - ❌ 從「庫存監控」打開
   - ❌ 從「已上架」列表打開

2. **資料狀態檢查** (`Model.ReviewStatus` 或 `Model.IsPending`)
   - ✅ 商品：ReviewStatus = 0 (待審核) 或 3 (重新申請)
   - ✅ 活動：Status = 0 (待審核) 或 4 (重新送審)
   - ❌ 商品：ReviewStatus = 1 (已核准)
   - ❌ 商品：ReviewStatus = 2 (已退回)
   - ❌ 活動：Status = 1 (已核准/進行中)
   - ❌ 活動：Status = 2 (已拒絕)
   - ❌ 活動：Status = 3 (已結束)

## 測試案例

### ✅ 應該顯示審核按鈕的情境
1. 從「商品審核中心」→「待審核」頁籤 → 點擊商品 → 應顯示「核准上架」和「退回」按鈕
2. 從「商品審核中心」→「重新申請審核」頁籤 → 點擊商品 → 應顯示「核准上架」和「再次駁回」按鈕
3. 從「促銷活動管理」→「待審核」頁籤 → 點擊活動 → 應顯示「核准」和「拒絕」按鈕
4. 從「促銷活動管理」→「重新申請審核」頁籤 → 點擊活動 → 應顯示「核准」和「拒絕」按鈕

### ❌ 不應顯示審核按鈕的情境
1. 從「商品總覽」→ 點擊商品 → **不顯示**審核按鈕，只顯示狀態資訊
2. 從「庫存監控」→ 點擊商品 → **不顯示**審核按鈕
3. 從「商品審核中心」→「近期審核通過」頁籤 → 點擊商品 → **不顯示**審核按鈕，顯示「已核准上架」資訊卡
4. 從「商品審核中心」→「近期退回紀錄」頁籤 → 點擊商品 → **不顯示**審核按鈕，顯示「已退回」資訊卡
5. 從「促銷活動管理」→「進行中」頁籤 → 點擊活動 → **不顯示**審核按鈕，顯示「已核准」資訊卡
6. 從「促銷活動管理」→「已拒絕」頁籤 → 點擊活動 → **不顯示**審核按鈕，顯示「已拒絕」資訊卡
7. 從「促銷活動管理」→「已結束」頁籤 → 點擊活動 → **不顯示**審核按鈕

## 維護注意事項

1. **新增審核頁面時**：記得在 JavaScript 調用時加上 `&isReviewMode=true` 參數
2. **新增詳情頁面時**：不要傳遞 `isReviewMode` 參數，讓它保持預設值 `false`
3. **修改狀態邏輯時**：請同步更新 `_shouldShowReviewButtons` 的條件判斷
4. **測試時**：請分別從審核中心和其他頁面打開同一筆資料，確認按鈕顯示正確
